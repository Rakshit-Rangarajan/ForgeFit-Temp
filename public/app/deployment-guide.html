FORGEFIT

Frontend on Vercel or Cloudflare Pages (free, global CDN). API and PostgreSQL on your laptop via Docker, exposed to the internet through a tunnel. No cloud server needed.

0

Architecture

Two completely separate pieces. The frontend is static HTML/CSS/JS — it costs nothing to host and serves instantly from a CDN. The API and database run locally on your laptop in Docker, made reachable from anywhere via a secure tunnel.

Internet (users, APK) │ ▼ ┌──────────────────────────────────────────┐ │ Vercel / Cloudflare Pages (FREE) │ │ forgefit.rakshitr.co.in │ │ │ │ / → landing page (index.html) │ │ /app/ → PWA app (app/index.html) │ │ /app/?watch=1 → watch layout │ └──────────────────────────────────────────┘ │ HTTPS API calls to absolute URL ▼ ┌──────────────────────────────────────────┐ │ ngrok / Cloudflare Tunnel │ │ api.forgefit.rakshitr.co.in │ │ (or https://abc123.ngrok-free.app) │ └──────────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────────┐ │ Your Laptop — Docker │ │ │ │ [forgefit_api] Node.js :3001 │ │ │ │ │ [forgefit_db] PostgreSQL :5432 │ │ │ (localhost only, not exposed) │ └──────────────────────────────────────────┘
💡
Why this setup?
You get a real public URL, global CDN performance, and zero hosting cost — while keeping your data on your own machine with no cloud database fees. The only catch: your laptop must be on and Docker must be running when you use the app.
⚠️
Laptop must be running
Whenever you want to use the app (login, log workouts, use AI coach), your laptop must be awake with Docker running and the tunnel active. The frontend works offline for reading, but all data operations need the API.

Expose API via Tunnel

Your API runs on localhost:3001. A tunnel creates a public HTTPS URL that forwards internet traffic to it. The frontend (on Vercel/CF Pages) needs this URL to make API calls.

Option A — Cloudflare Tunnel (Recommended — Permanent URL)

1
Install cloudflared
# Mac:
brew install cloudflare/cloudflare/cloudflared

# Windows (PowerShell as admin):
winget install --id Cloudflare.cloudflared

# Linux:
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cf.deb
sudo dpkg -i cf.deb
2
Log in to Cloudflare
cloudflared tunnel login
# Opens browser — sign in to your Cloudflare account
# Select the zone: rakshitr.co.in
3
Create a named tunnel
cloudflared tunnel create forgefit-api
# Output: Tunnel credentials written to ~/.cloudflared/<UUID>.json
# Note the UUID — you need it in the next step
4
Create the tunnel config file
Create ~/.cloudflared/config.yml:
tunnel: <YOUR-TUNNEL-UUID>
credentials-file: /Users/YOUR_USERNAME/.cloudflared/<UUID>.json

ingress:
  - hostname: api.forgefit.rakshitr.co.in
    service: http://localhost:3001
  - service: http_status:404
5
Add DNS record in Cloudflare
cloudflared tunnel route dns forgefit-api api.forgefit.rakshitr.co.in
# Creates a CNAME: api.forgefit.rakshitr.co.in → <UUID>.cfargotunnel.com
6
Run the tunnel
cloudflared tunnel run forgefit-api
# API is now live at: https://api.forgefit.rakshitr.co.in
# Keep this terminal open while using the app
7
Auto-start tunnel on login (optional)
# Mac — run as a service:
sudo cloudflared service install
sudo launchctl start com.cloudflare.cloudflared

# Linux (systemd):
sudo cloudflared service install
sudo systemctl start cloudflared
sudo systemctl enable cloudflared

Option B — ngrok (Quicker setup, URL changes on free plan)

1
Install and authenticate
# Mac:
brew install ngrok/ngrok/ngrok
# Sign up at https://ngrok.com, then:
ngrok config add-authtoken YOUR_AUTH_TOKEN
2
Start the tunnel
ngrok http 3001
# Shows your public URL, e.g.:
Forwarding  https://abc123.ngrok-free.app → localhost:3001
3
Update your Vercel API_URL env var
Each time ngrok restarts (free plan), you get a new URL. Update the API_URL env var in your Vercel dashboard and redeploy. On paid ngrok plans you can pin a static subdomain to avoid this.
🚀
Quick start script
The project includes a helper: ./scripts/start-tunnel.sh ngrok or ./scripts/start-tunnel.sh cloudflare

Deploy Frontend to Vercel

Vercel detects the project as a static site automatically. It serves your HTML from a global CDN with zero configuration.

1
Push project to GitHub
cd ~/Documents/forgefit
git init
git add .
git commit -m "Initial commit — ForgeFit"
# Create repo at github.com, then:
git remote add origin https://github.com/YOURNAME/forgefit.git
git push -u origin main
⚠️
.env is excluded automatically
.gitignore already excludes .env and nginx/certs/. Never manually add them.
2
Import project on Vercel
Go to vercel.com → New Project → Import from GitHub → select your forgefit repo.
3
Set build settings
SettingValue
Framework PresetOther
Root Directory./ (repo root)
Build Commandnode scripts/inject-api-url.js
Output Directoryfrontend/public
Install Commandleave blank
4
Add the API_URL environment variable
Still on the Vercel project setup page, scroll to Environment Variables:
NameValue
API_URLhttps://api.forgefit.rakshitr.co.in (or your ngrok URL)
5
Click Deploy
Vercel builds in ~30 seconds. You get a URL like forgefit.vercel.app. Then add your custom domain:
Project Settings → Domains → Add Domain
Type: forgefit.rakshitr.co.in
Follow Vercel's instructions to add a CNAME in your DNS
6
Add your Vercel domain to .env FRONTEND_URL
Open your local .env and set:
FRONTEND_URL=https://forgefit.rakshitr.co.in
Then restart the API container:
docker compose restart api
7

Updating API URL in Vercel

Any time your tunnel URL changes (ngrok free plan restarts), you need to update the environment variable in Vercel and redeploy.

# Quick way via Vercel CLI:
npm i -g vercel
vercel env add API_URL production
# Paste new tunnel URL when prompted
vercel --prod  # redeploy
💡
Use Cloudflare Tunnel to avoid this
With a named Cloudflare Tunnel, your API URL is always https://api.forgefit.rakshitr.co.in — it never changes. Set API_URL in Vercel once and never touch it again.

Deploy to Cloudflare Pages

Alternative to Vercel. Also free, also global CDN. Works especially well if you're already using Cloudflare for DNS (which you are, if you use Cloudflare Tunnel).

1
Push to GitHub (same as Vercel step 1)
Same Git setup — skip this if already done.
2
Create a Cloudflare Pages project
Go to dash.cloudflare.com → Pages → Create a project → Connect to Git → select your forgefit repo.
3
Build settings
SettingValue
Production branchmain
Framework presetNone
Build commandnode scripts/inject-api-url.js
Build output directoryfrontend/public
4
Add environment variable
In the Cloudflare Pages project → Settings → Environment Variables → Production:
API_URL = https://api.forgefit.rakshitr.co.in
5
Custom domain
Pages → Custom Domains → Add forgefit.rakshitr.co.in. Since your DNS is already on Cloudflare, this activates instantly — no CNAME needed.

Common Errors

CORSAccess to fetch at 'https://api...' from origin 'https://forgefit...' has been blocked
The API's CORS config doesn't include your frontend URL.
Fix
# In your .env on the laptop:
FRONTEND_URL=https://forgefit.rakshitr.co.in
# Then:
docker compose restart api
APIFailed to fetch / Network Error when calling API from frontend
Tunnel is not running, or Docker API container is down.
Fix
# Check Docker:
docker compose ps
docker compose logs api --tail=20

# Check tunnel:
curl https://api.forgefit.rakshitr.co.in/health

# Restart tunnel:
cloudflared tunnel run forgefit-api
# or: ngrok http 3001
VercelApp loads but API URL still shows localhost or wrong URL
The inject-api-url.js script didn't find the placeholder, or API_URL env var wasn't set before deploy.
Fix
# In Vercel dashboard: Settings → Environment Variables
# Add or update: API_URL = https://api.forgefit.rakshitr.co.in
# Then: Deployments → Redeploy (top-right menu on latest deploy)
DBLogin returns "Invalid credentials"
Typo when inserting user, or DB was wiped.
Fix
docker exec -it forgefit_db psql -U forgefit_user -d forgefit
SELECT username, is_active FROM users;
UPDATE users SET password_hash = crypt('NewPass', gen_salt('bf')) WHERE username = 'rakshit';
Cloudflaretunnel: failed to connect to origin — connection refused
Docker API container not running when tunnel tried to connect.
Fix
# Start Docker first, then start the tunnel:
docker compose up -d
cloudflared tunnel run forgefit-api
